[[overview]]
== 개요

[[overview-common-response]]
=== 공통 응답 구조

==== 성공 응답
[source,json]
----
{
  "data": { ... },
  "message": "성공 메시지"
}
----

==== 에러 응답

NOTE: 테스트 환경에선 `traceId`가 포함되지 않아 API 문서에는 포함되지 않습니다.

.일반 비즈니스 에러
[source,json]
----
{
  "traceId": "a1b2c3d4",
  "message": "만료된 Refresh Token입니다."
}
----

.Validation 에러
[source,json]
----
{
  "traceId": "a1b2c3d4",
  "message": "에러 메시지",
  "errors": [
    {
      "field": "필드명",
      "value": "입력값",
      "reason": "에러 사유"
    }
  ]
}
----

[[overview-authentication]]
=== 인증 방식

본 API는 JWT 기반 인증을 사용합니다.

* 인증이 필요한 API는 아래 헤더를 반드시 포함해야 합니다.

[source,http]
----
Authorization: Bearer {accessToken}
----

* 인증이 필요하지 않은 API는 각 API 설명에 별도로 명시합니다.

[[overview-pagination]]
=== 페이지네이션

NOTE: 페이지네이션 파라미터 이름/기본값은 API에 따라 다를 수 있으며, 해당 API의 `query-parameters` 스니펫을 우선합니다.

==== 요청 파라미터
[cols="1,1,1,3"]
|===
|파라미터 |타입 |필수 |설명

|pageNo
|Integer
|X
|페이지 번호 (0부터 시작하는 API도 존재할 수 있음)

|pageSize
|Integer
|X
|페이지 크기 (기본값/최대값은 API별 상이)
|===

==== 응답 구조
[source,json]
----
{
  "data": {
    "content": [
      {
        "id": 1,
        "name": "상품명"
      }
    ],
    "pagination": {
      "currentPage": 1,
      "totalPages": 5,
      "totalItems": 47,
      "pageSize": 10,
      "first": true,
      "last": false,
      "hasNext": true,
      "hasPrevious": false
    }
  },
  "message": "성공 메시지"
}
----

==== 응답 필드
[cols="1,1,3"]
|===
|필드 |타입 |설명

|content
|Array
|현재 페이지의 데이터

|pagination
|Object
|페이지 정보

|pagination.currentPage
|Integer
|현재 페이지 번호

|pagination.totalPages
|Integer
|전체 페이지 수

|pagination.totalItems
|Long
|전체 아이템 수

|pagination.pageSize
|Integer
|페이지당 아이템 수

|pagination.first
|Boolean
|첫 페이지 여부

|pagination.last
|Boolean
|마지막 페이지 여부

|pagination.hasNext
|Boolean
|다음 페이지 존재 여부

|pagination.hasPrevious
|Boolean
|이전 페이지 존재 여부
|===
